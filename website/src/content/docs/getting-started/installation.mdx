## System Requirements

- **Node.js**: 18.0.0 or higher
- **npm**: 7.0.0 or higher (comes with Node.js)
- **Git**: 2.0.0 or higher (recommended for versioned docs)

Check your versions:

```bash
node --version  # Should be v18.0.0 or higher
npm --version   # Should be 7.0.0 or higher
git --version   # Should be 2.0.0 or higher
```

## Installation Methods

<Tabs>
  <TabItem label="npm">
    **Install globally** to use across all projects:

    ```bash
    npm install -g cortex-tms@3.1.0
    ```

    **Verify installation**:

    ```bash
    cortex-tms --version
    # Output: 3.1.0
    ```

    **Usage**:

    ```bash
    cd your-project
    cortex-tms init
    ```

    **Pros**: Use `cortex-tms` command anywhere, fastest execution (no download on each run)

    **Cons**: Requires global install permissions, may conflict with other versions
  </TabItem>

  <TabItem label="npx">
    **Run without installing** globally:

    ```bash
    npx cortex-tms@3.1.0 init
    ```

    **Verify** (will download and run):

    ```bash
    npx cortex-tms@3.1.0 --version
    # Output: 3.1.0
    ```

    **Pros**: No global installation required, always run specific version, no permission issues

    **Cons**: Slower (downloads on first run per session), must specify version each time

    <Aside type="tip" title="Create an Alias">
      Add to `~/.bashrc` or `~/.zshrc`:

      ```bash
      alias cortex="npx cortex-tms@3.1.0"
      ```

      Then use: `cortex init`
    </Aside>
  </TabItem>

  <TabItem label="pnpm">
    **Install with pnpm** as your package manager:

    ```bash
    pnpm add -g cortex-tms@3.1.0
    ```

    **Usage**:

    ```bash
    cortex-tms init
    ```

    **Pros**: Faster installs with pnpm's symlink strategy, disk space efficient, monorepo-friendly
  </TabItem>

  <TabItem label="Yarn">
    **Install with Yarn** as your package manager:

    ```bash
    yarn global add cortex-tms@3.1.0
    ```

    **Usage**:

    ```bash
    cortex-tms init
    ```

    <Aside type="caution">
      Ensure Yarn's global bin is in your PATH:

      ```bash
      export PATH="$(yarn global bin):$PATH"
      ```
    </Aside>
  </TabItem>
</Tabs>

After installation, verify Cortex TMS is working:

```bash

# View help
cortex-tms --help

# Test initialization (in a test directory)
```

## Troubleshooting Installation

<Aside type="caution" title="Command not found: cortex-tms">
  **Cause**: npm's global bin directory is not in your PATH.

  **Solution**:

  1. Find npm's global bin path:
     ```bash
     npm config get prefix
     ```

  2. Add it to your PATH (Linux/macOS):
     ```bash
     echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
     source ~/.bashrc
     ```

  3. For Windows PowerShell:
     ```powershell
     $env:Path += ";$(npm config get prefix)"
     ```
</Aside>

<Aside type="caution" title="Permission denied when installing globally">
  **Cause**: npm lacks permissions to install global packages.

  **Solution 1**: Use npx instead:

  ```bash
  npx cortex-tms@3.1.0 init
  ```

  **Solution 2**: Configure npm to use a user directory:

  ```bash
  mkdir ~/.npm-global
  npm config set prefix '~/.npm-global'
  echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
  source ~/.bashrc

  # Now install globally
  npm install -g cortex-tms@3.1.0
  ```
</Aside>

<Aside type="caution" title="Version mismatch or outdated version">
  **Cause**: Cached old version or multiple installations.

  **Solution**:

  1. Uninstall all versions:
     ```bash
     npm uninstall -g cortex-tms
     ```

  2. Clear npm cache:
     ```bash
     npm cache clean --force
     ```

  3. Reinstall specific version:
     ```bash
     npm install -g cortex-tms@3.1.0
     ```
</Aside>

<Aside type="caution" title="Windows-specific issues">
  **Issue**: Scripts disabled in PowerShell.

  **Solution**: Enable script execution:

  ```powershell
  Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
  ```
</Aside>

## Updating Cortex TMS

To update to the latest version:

```bash
# Check current version
cortex-tms --version

# Update to latest
npm update -g cortex-tms

# Or install specific version
npm install -g cortex-tms@3.1.0
```

## Uninstalling

To remove Cortex TMS:

```bash
npm uninstall -g cortex-tms
```

Your project files created by Cortex TMS (like `CLAUDE.md`, `NEXT-TASKS.md`) will remain untouched.
