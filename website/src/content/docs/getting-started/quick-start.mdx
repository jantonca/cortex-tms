This guide will have you up and running with Cortex TMS in under 5 minutes.

## Prerequisites

- **Node.js 18+** installed
- A project directory (new or existing)
- _(Optional)_ Claude Code or GitHub Copilot for AI assistance

## Installation & Setup

<Steps>

1. **Install Cortex TMS**

   Choose your preferred installation method:

   <Tabs>
     <TabItem label="npm">
       ```bash
       npm install -g cortex-tms@3.1.0
       ```
     </TabItem>

     <TabItem label="npx (no install)">
       ```bash
       npx cortex-tms@3.1.0 init
       ```
     </TabItem>

     <TabItem label="pnpm">
       ```bash
       pnpm add -g cortex-tms@3.1.0
       ```
     </TabItem>

     <TabItem label="yarn">
       ```bash
       yarn global add cortex-tms@3.1.0
       ```
     </TabItem>
   </Tabs>

   <Aside type="tip" title="Global Install Benefits">
     Installing globally unlocks the **`cortex` shorthand command** for faster workflows:

     ```bash
     # Instead of typing:
     cortex-tms status
     cortex-tms review src/file.ts

     # You can use:
     cortex status
     cortex review src/file.ts
     ```

     This is especially useful for frequent commands and prepares you for v2.8 cross-project features like `cortex list`.
   </Aside>

2. **Initialize your project**

   Navigate to your project directory and run:

   ```bash
   cd your-project
   cortex init
   # Or: cortex-tms init (both commands work identically)
   ```

   You'll be prompted to choose:
   - **Project type**: Basic, Full, or Custom
   - **Features**: ADRs, Patterns, Glossary, etc.

   <Aside type="tip" title="Recommended">
     Select **"Full"** to get all features for this tutorial.
   </Aside>

3. **Review generated files**

   Cortex TMS creates this structure:

   <FileTree>
   - your-project/
     - CLAUDE.md AI agent instructions
     - NEXT-TASKS.md Current objectives
     - CHANGELOG.md Version history
     - docs/
       - core/
         - PATTERNS.md Code conventions
         - GLOSSARY.md Terminology
       - adr/
         - 0001-example-adr.md
     - .github/
       - copilot-instructions.md
   </FileTree>

4. **Customize for your project**

   Open `CLAUDE.md` and update:
   - **Role**: Your AI assistant's persona
   - **CLI Commands**: Add your test/build commands
   - **Operational Loop**: Modify the workflow if needed

   Example customization:

   ```markdown
   ## ðŸ’» CLI Commands

   - **Test**: `npm test` or `pytest`
   - **Lint**: `npm run lint`
   - **Build**: `npm run build`
   ```

5. **Test with Claude Code** (if installed)

   Open your project in Claude Code:

   ```bash
   claude-code your-project
   ```

   Claude will automatically read `CLAUDE.md` and follow your instructions!

</Steps>

## What's Next?

### Update NEXT-TASKS.md

Edit `NEXT-TASKS.md` to add your first real task:

```markdown
## Current Objective

Implement user authentication using JWT tokens.

**Why**: We need secure API access for the mobile app.

**Acceptance Criteria**:
- [ ] JWT token generation on login
- [ ] Token validation middleware
- [ ] Refresh token rotation
- [ ] Tests for auth flow
```

### Create Your First ADR

When you make an architectural decision, document it:

```bash
cortex adr new "Use PostgreSQL for user data storage"
```

This creates `docs/adr/0002-use-postgresql-for-user-data-storage.md` with a template.

### Codify Your Patterns

Edit `docs/core/PATTERNS.md` to add conventions:

```markdown
## File Naming

- React components: `PascalCase.tsx`
- Utility functions: `camelCase.ts`
- Test files: `*.test.ts`
```

## Using with AI Agents

<Tabs>
  <TabItem label="Claude Code">
    Claude Code reads `CLAUDE.md` automatically. Just open your project:

    ```bash
    claude-code .
    ```

    <Aside type="tip">
      Claude will automatically follow your instructions and workflows defined in `CLAUDE.md`.
    </Aside>
  </TabItem>

  <TabItem label="GitHub Copilot">
    Copilot reads `.github/copilot-instructions.md`. Cortex TMS generates this with a reference to your main docs:

    ```markdown
    See CLAUDE.md for full project conventions.
    ```

    <Aside type="note">
      GitHub Copilot uses the `.github/copilot-instructions.md` file to understand your project conventions.
    </Aside>
  </TabItem>

  <TabItem label="Cursor">
    Cursor supports `.cursorrules` files. You can copy `CLAUDE.md` content there:

    ```bash
    cp CLAUDE.md .cursorrules
    ```

    <Aside type="tip">
      Keep both files in sync by creating a symlink: `ln -s CLAUDE.md .cursorrules`
    </Aside>
  </TabItem>
</Tabs>

## Troubleshooting

<Aside type="caution" title="Command not found: cortex-tms or cortex">
  Ensure npm global bin is in your PATH:

  ```bash
  npm config get prefix
  ```

  The output should be in your `$PATH`. If not, add it:

  ```bash
  export PATH="$(npm config get prefix)/bin:$PATH"
  ```

  Both `cortex` and `cortex-tms` commands should work after global installation.
</Aside>

<Aside type="caution" title="Validation failed: NEXT-TASKS.md malformed">
  Check that `NEXT-TASKS.md` has:
  - A `## Current Objective` heading
  - At least one task description
  - Proper Markdown formatting

  Run `cortex validate --help` for detailed error messages.
</Aside>
