## Technical Decisions

### Decision: Mirror Destination Structure in `templates/`
**Context**: Initially considered separate mapping logic
**Decision**: `templates/` now mirrors the exact structure of destination projects
**Rationale**: Reduces cognitive load for CLI tool implementation (no mapping translation needed)
**Impact**: Simplified Phase 4 CLI development

### Decision: Lightweight Validation (Option A)
**Context**: Phase 2 validation could be thorough (full sandbox app) or lightweight (validation doc)
**Decision**: Created `docs/validation/phase2-sandbox.md` instead of `examples/sandbox-app/`
**Rationale**: Phase 3's `todo-app` will provide comprehensive real-world validation
**Impact**: Accelerated Phase 3 start date
